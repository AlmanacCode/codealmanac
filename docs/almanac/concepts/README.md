# Concepts

`concepts/` explains the vocabulary a new maintainer needs before the codebase
makes sense. Put stable project terms here when they cut across several files or
subsystems.

Good concept pages explain what the term means in this repo, where it appears,
which pages depend on it, and what assumptions surround it.
