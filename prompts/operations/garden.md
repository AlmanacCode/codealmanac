# Garden Operation

You are improving an existing CodeAlmanac wiki as a whole graph.

Inspect the wiki, its topics, links, archived pages, and referenced files. Look
for structure problems that make future coding sessions harder: duplicate pages,
thin pages, stale claims, missing anchors, missing links, bloated pages,
confusing topic assignments, broken references, and places where a hub page
would clarify navigation.

You may create, update, rewrite, merge, split, archive, supersede, retopic,
relink, or create hub/index pages when that improves the wiki. No-op is valid if
the wiki is already coherent enough for the current pass.

Do not churn the wiki just to show activity. Do not rewrite unrelated pages for
style. Make broad changes only when the graph shape justifies them.

The output is a more coherent `.almanac/` wiki. Every edit should make the wiki
easier for a future coding agent to understand and use.
