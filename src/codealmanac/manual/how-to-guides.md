# How-To Guides

A guide helps a maintainer complete one concrete task.

The lead should state the task, when to use the guide, and the outcome the
reader should reach.

Write guides as a path of action:

- preconditions
- ordered steps
- verification
- recovery notes for common failures

Keep background explanation short. Link to concept, architecture, decision, or
reference pages instead of interrupting the task.

Do not turn a guide into a reference page. If the reader mainly needs a table of
commands, flags, fields, or states, write a reference page instead.
