# Reference

A reference page is exact lookup material.

The lead should define the scope of the reference and what a reader can look up
there.

Use reference pages for commands, flags, schemas, frontmatter, config keys,
states, models, file layout, stable behavior, and tables of exact contracts.

Reference pages should be neutral, scannable, and organized by the structure of
the thing being described. Use lists or tables when they make exact behavior
easier to compare.

Split reference pages by lookup need. A command surface, schema, state enum,
frontmatter format, file layout, and event format are usually separate
references because readers look them up at different moments.

Do not turn a reference page into a tutorial or a motivation essay.
